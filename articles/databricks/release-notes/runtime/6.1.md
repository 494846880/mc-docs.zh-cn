# <a name="databricks-runtime-61-unsupported"></a>Databricks Runtime 6.1（不受支持）

Databricks 于 2019 年 10 月发布了此映像。

以下发行说明提供了由 Apache Spark 支持的 Databricks Runtime 6.1 的相关信息。

## <a name="new-features"></a>新增功能

### <a name="delta-lake-enhancements"></a>Delta Lake 增强功能

#### <a name="python-apis-for-delta-tables-public-preview"></a>适用于 Delta 表的 Python API（公共预览版）

现在可以使用 Python API 在 Delta 表中删除和更新插入数据、清理表并获取表历史记录。

有关详细信息，请参阅[表删除、更新和合并](../../delta/delta-update.md)以及[表实用工具命令](../../delta/delta-utility.md)。

#### <a name="dynamic-file-pruning-dfp-enabled-by-default"></a>默认情况下已启用的动态文件修剪 (DFP)

DFP 使用 Delta 表元数据（例如，文件中的最小/最大列统计信息），通过跳过列值超出范围的文件，大大提高了许多查询的性能。 与动态分区修剪 (DPP) 不同，DFP 也适用于非分区列。
对于在非分区键上进行联接的工作负荷，DFP 可能非常有用。

### <a name="configure-clusters-with-your-own-container-image-using-databricks-container-services"></a>通过 Databricks 容器服务使用自己的容器映像配置群集

Databricks 容器服务允许你使用你自己的容器映像配置群集。
可以将复杂环境预打包在容器中，将其发布到常用容器注册表，如 ACR、ECR 或 Docker Hub，然后让 Databricks 拉取映像以构建群集。 一些示例用例包括：

* 库自定义 - 你可以完全控制你要安装的系统库
* 黄金容器环境 - 你的 Docker 映像是锁定的环境，永远不会更改
* Docker CI/CD 集成 - 可以将 Azure Databricks 与 Docker CI/CD 管道集成

还有许多其他用例，涵盖从“指定配置”到“安装机器学习包”等许多方面。

有关详细信息，请参阅[使用 Databricks 容器服务自定义容器](../../clusters/custom-containers.md#containers)。

### <a name="secrets-referenced-by-spark-configuration-properties-and-environment-variables-public-preview"></a>Spark 配置属性和环境变量引用的机密（公共预览版）

自 Databricks Runtime 6.1 的 [11 月 7 日维护更新](maintenance-updates.md#61)起，在 Spark 配置属性或环境变量中引用[机密](../../security/secrets/index.md)的功能以公开预览版提供。 有关详细信息，请参阅 [Spark 配置属性和环境变量中的机密路径](../../security/secrets/secrets.md#spark-conf-env-var)。

### <a name="ganglia-metrics-now-available-on-clusters-with-table-acls-and-credential-passthrough"></a>Ganglia 指标现已在具有表 ACL 和凭据直通验证功能的群集上提供

启用了[表访问控制](../../security/access-control/table-acls/object-privileges.md)或[凭据直通身份验证](../../security/credential-passthrough/adls-passthrough.md)的群集以前不支持 Ganglia 指标。 现在它们已经可用。

## <a name="improvements"></a>改进

* 已将 netty-all 从 4.1.17.Final 升级到 4.1.39.Final
* 已将 Snowflake 连接器从 2.4.10 升级到 2.5.3

### <a name="credential-passthrough"></a>凭据直通身份验证

* Azure Active Directory 凭据现在会在使用 `dbutils.notebook.run()` 运行的笔记本中传播。
* 所有 PySpark ML 类现在都受支持。
* Ganglia UI 现已可用。

有关详细信息，请参阅[使用 Azure Active Directory 凭据直通身份验证保护对 Azure Data Lake Storage 的访问](../../security/credential-passthrough/adls-passthrough.md)。
